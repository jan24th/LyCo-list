# 实现任务移动、完成、恢复和乐观并发

## User Story

作为用户，我希望移动、完成和恢复任务，并检测冲突，以便跨设备安全地修改任务。

## Acceptance Criteria

### Scenario 1: 移动任务到另一个列表

Given 列表 A 中有一个任务
When 用户将其移动到列表 B
Then 任务属于列表 B

### Scenario 2: 完成任务

Given 一个活跃的任务
When 用户将其标记为完成
Then 任务被完成并记录时间戳

### Scenario 3: 恢复已完成的任务

Given 一个已完成的任务
When 用户恢复它
Then 任务重新变为活跃状态

### Scenario 4: 乐观并发冲突

Given 任务 version 为 2
When 发送 expectedVersion 为 1 的更新
Then API 返回 409
