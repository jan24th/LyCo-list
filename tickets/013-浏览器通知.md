# 浏览器通知

## User Story

As a 用户，I want 在任务到期时收到浏览器通知，So that 即使应用未打开我也不会错过重要截止日期。

## Acceptance Criteria

### Scenario 1: 请求通知权限

Given 用户安装 PWA 或启用通知
When 应用请求通知权限
Then 用户可以选择授予或拒绝

### Scenario 2: 前端调度器从后端获取即将到来提醒

Given 应用在线
When 前端调度器每小时调用 `GET /api/reminders/upcoming`
Then 获取到未来触发的提醒列表并缓存到 Service Worker 或 IndexedDB

### Scenario 3: 触发本地通知

Given PWA 已安装且权限已授予
When 提醒触发时间到达
Then 浏览器显示包含任务标题和时间的通知

### Scenario 4: 补检逾期提醒

Given 应用重新打开
When 前端检查逾期提醒
Then 应用内展示逾期提醒或徽章

### Scenario 5: 通知精确度说明

Given Web 通知存在限制
When 用户查看通知设置
Then 看到"通知为尽力而为，可能延迟"的说明

### Scenario 6: TDD 开发与 100% 覆盖率

Given 项目要求 TDD
When 为通知调度和权限逻辑编写测试
Then 测试先失败，实现后通过，覆盖率达到 100%
