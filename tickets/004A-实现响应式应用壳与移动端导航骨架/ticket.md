# 实现响应式应用壳与移动端导航骨架

## 用户故事

作为移动端用户，我希望应用在小屏设备上提供适合触控的导航与内容布局，以便后续功能页面无需重复补做全局移动端适配。

## 背景

设计文档要求桌面端使用固定左侧边栏，移动端使用顶部标题栏与汉堡抽屉。已归档的 ticket 004 只建立了 React 路由与基础样式，没有提供响应式应用壳。该能力必须在列表、任务和搜索页面之前建立，作为后续前端 ticket 的共同布局基础。

## 范围

### 包含

- 建立可复用的响应式应用壳，接收页面标题、导航内容和主内容插槽
- 视口宽度达到 Tailwind `lg` 断点时显示固定左侧导航区域，隐藏移动端标题栏
- 视口宽度低于 Tailwind `lg` 断点时隐藏固定侧边栏，显示顶部标题栏与汉堡按钮
- 使用 004B 提供的 shadcn/ui `Sheet` 承载移动端导航内容
- 点击移动端导航项后关闭抽屉，并支持 Escape、遮罩点击和关闭按钮
- 主内容区在移动端全宽展示，不产生页面级水平滚动
- 支持移动设备安全区域，并保证主要触控目标最小尺寸为 44 × 44 CSS px
- 视口跨越断点或设备横竖屏切换时，布局自动切换

### 不包含

- 智能列表、自定义列表和“新建列表”等业务导航内容（008A）
- 列表、任务、提醒、搜索等业务页面内部的响应式排版
- 滑动手势打开或关闭抽屉
- 平板端独立布局；Tailwind `lg` 断点是桌面与移动布局的唯一边界
- PWA manifest、Service Worker 和安装提示（016）

## 验收标准

### 场景 1：桌面端应用壳

Given 视口宽度达到 Tailwind `lg` 断点
When 用户打开任一应用页面
Then 固定左侧导航区域与主内容区可见，移动端标题栏和汉堡按钮隐藏

### 场景 2：移动端应用壳

Given 视口宽度低于 Tailwind `lg` 断点
When 用户打开任一应用页面
Then 固定侧边栏隐藏，顶部标题栏显示当前页面标题与汉堡按钮，主内容区全宽展示

### 场景 3：打开和关闭移动端导航

Given 用户处于移动端布局
When 用户点击汉堡按钮
Then shadcn/ui Sheet 打开并展示传入的导航内容
And 用户可通过 Escape、遮罩或关闭按钮关闭抽屉

### 场景 4：导航后自动关闭

Given 移动端导航抽屉已打开
When 用户选择任一导航项
Then 页面完成导航且抽屉关闭

### 场景 5：适配移动设备边界

Given 应用运行在带安全区域的移动设备上
When 页面渲染顶部标题栏和主内容区
Then 内容不被系统区域遮挡，主要触控目标不小于 44 × 44 CSS px，页面不出现水平滚动

### 场景 6：断点变化自动切换

Given 应用已经打开
When 视口跨越 Tailwind `lg` 断点或设备方向变化
Then 应用壳自动切换对应布局且保留当前路由内容

## 测试要求

- 对响应式 class、安全区域和最小触控尺寸进行结构断言
