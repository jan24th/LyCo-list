# 后端脚手架 Hono + Prisma + SQLite

## User Story

As a 后端开发者，I want 一个可运行的 Hono 服务，So that 可以在此之上实现 REST API 和业务逻辑。

## Acceptance Criteria

### Scenario 1: 初始化 Hono 项目

Given `apps/api` 目录已存在
When 安装 Hono 并创建入口文件
Then 运行 `bun dev` 后服务在指定端口启动

### Scenario 2: 配置环境变量

Given 后端服务需要数据库连接和端口
When 创建 `.env` 与 `.env.example` 文件
Then 应用可以读取 `DATABASE_URL` 和 `PORT` 配置

### Scenario 3: 配置 Zod 校验

Given 后端需要校验请求数据
When 安装 Zod 并创建共享校验 schema
Then 路由可以使用 Zod 校验请求体

### Scenario 4: 配置测试环境

Given 项目要求 TDD
When 在 `apps/api` 中配置 Vitest 和测试脚本
Then 可以运行 `bun test` 并看到测试通过

### Scenario 5: 添加健康检查接口

Given 服务已启动
When 访问 `/api/health`
Then 返回 `200 OK` 和状态信息

### Scenario 6: 配置 SQLite 数据库连接

Given 后端服务已启动
When 配置 Prisma 的 `DATABASE_URL` 指向 SQLite 文件并执行首次连接
Then 应用可以成功连接并查询 SQLite 数据库
